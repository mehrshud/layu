# Contributing to layu
=======================

We welcome contributions to layu from the community. Here's a step-by-step guide on how to contribute:

## Forking the Repository
-------------------------

To contribute to layu, you'll need to fork the repository. Here's how:

1. Go to the [layu repository](https://github.com/layu/layu) on GitHub.
2. Click the "Fork" button in the top-right corner of the repository.
3. Choose the account where you want to fork the repository.
4. Wait for the repository to be forked.

## Branch Naming
----------------

When creating a new branch for your contribution, please use the following naming convention:

* `feature/<feature-name>` for new feature branches
* `fix/<fix-description>` for bug fix branches
* `docs/<docs-description>` for documentation changes
* `refactor/<refactor-description>` for code refactoring

## Conventional Commits
-----------------------

We use [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) to ensure consistent commit messages. Please format your commit messages as follows:

* `type: brief description`
	+ `type` is one of: `feat`, `fix`, `docs`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`
	+ `brief description` is a short summary of the changes made

Example:
```markdown
feat: add new API endpoint for user management
```

## Pull Request Checklist
-------------------------

Before submitting a pull request, please ensure that:

* You've forked the repository and created a new branch for your changes
* Your branch is up-to-date with the latest changes from the main repository
* You've formatted your commit messages according to the Conventional Commits specification
* You've tested your changes thoroughly
* You've included a clear and concise description of your changes in the pull request
* You've checked that your changes do not introduce any new bugs or issues

By following these guidelines, you'll help us review and merge your contributions more efficiently. Thank you for contributing to layu!
